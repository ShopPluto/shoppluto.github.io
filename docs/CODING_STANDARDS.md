# 💻 Pluto Coding Standards

Version: 1.0

---

# Purpose

This document defines the coding standards for the Pluto project.

Every HTML, CSS and JavaScript file must follow these rules.

---

# General Principles

- Write clean code.
- Write readable code.
- Avoid duplication.
- Keep files modular.
- Use meaningful names.

---

# File Naming

Use lowercase.

Use hyphens.

✅ Correct

product-card.html

shopping-cart.js

product-grid.css

❌ Wrong

ProductCard.html

MyFile.css

final.js

---

# HTML Rules

- Use semantic HTML.
- Indent using 4 spaces.
- Always include alt text for images.
- Use meaningful class names.
- Avoid inline CSS.
- Avoid inline JavaScript.

---

# CSS Rules

- Use CSS variables.
- Mobile First.
- Keep selectors simple.
- Group related styles.
- Avoid !important unless necessary.

---

# JavaScript Rules

- Use const by default.
- Use let only when values change.
- Never use var.
- Use meaningful function names.
- Keep functions small.

---

# Comments

Write comments only when they explain why, not what.

---

# Git Commits

Good examples

Add homepage hero section

Create product card component

Implement responsive navbar

Bad examples

Update

Done

Final

---

# Project Goal

Every developer working on Pluto should be able to understand the code quickly.
